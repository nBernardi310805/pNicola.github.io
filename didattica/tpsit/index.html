Come è composto un sito web

Premessa:

Imparare a creare un sito web non è come imparare a disegnare, cosa che tra l'altro non so fare, ma è più complesso perché bisogna imparare i codici descrittivi (HTML, CSS) e possibilmente il JavaScript.

Per creare un sito web

L'HTML

L'HTML è un linguaggio descrittivo. Serve per creare il sito web nella sua struttura, quindi titoli, testo, immagini

L'iframe sovrastante mostra la Home Page del mio sito senza CSS o JavaScript.

cc

Il CSS

Il CSS è il linguaggio descrittivo che serve a dare una grafica al sito.

Il JavaScript

Il JavaScript serve a rendere il sito interattivo. Ha una sintassi simile a quella del C.

Perché non basta "rubare" un template fare un sito?

Perché non ha senso fare un sito basandosi solo su un template perché si è limitati dalla grafica, dal vecchio contenuto del modello, ecc.

Creazione di un sito - cose essenziali

Consigli

  • Avere un'idea.

Sembra una cosa banale e scontata, ma il mio primo consiglio sulla creazione di siti è inanzitutto avere un'idea precisa riguardo ai contenuti o allo scopo del sito. Questo primo consiglio è collegato agli altri, come vedrai tra poco.

  • Mantenere ordine

La creazione di un sito, per chi non lo sapesse, avviene tramite una serie di codici, e linguaggi di programmazione. È quindi molto importante (per lo sviluppatore) mantenere ordine tra le righe e formattare il documento. Per programmare vi consiglio Visual Studio Code (con alcune estensioni). Usare il blocco note o un qualsiasi altro editor di testo non fatto apposta per il coding vi farà diventare pazzo (parlo per esperienza).

Vi consiglio di usare divisori semantici per ordinare il documento HTML. Un esempio sono article, section, header, footer... Un divisore non semantico è il vecchio div.

  • Scegliere il template giusto

Anche questa è abbastanza banale e si collega con il primo consiglio (quello dell'idea). Non è necessario creare un template da capo, come ho fatto io, ma è necessario conoscere un minimo di CSS, in caso qualcosa del template già fatto non vi piaccia.

Potete usare il template del mio sito per farlo incolla la seguente riga di codice tra i tag <head> e </head>:

<link rel="stylesheet" href="https://portalenicola.it/css/core.css">

Interazione HTML - JavaScript

L'interazione tra HTML e JS avviene usando alcuni codici che ora vedremo.

Come faccio a fare uno spazio di input?

Lo spazio input si fa con l'HTML. Quello che faremo oggi non sarà altro che fare alcuni campi di input (prima indipendenti) con l'HTML e poi dire alla nostra funzione JS di prendersi il valore. Quindi:

P.s. Create poi sotto due caselle di testo con id usr e pass

    <label for="username">Inserisci l'username<label>
    <input type="text" id="username"><br>
    <label for="password">Inserisci la password<label>
    <input type="password" id="password">
    <input type="submit" onclick="accedi()">  

<p id="usr"></p>
<p id="pass"></p>

Ok, ora andiamo a creare la nostra funzione JavaScript

function accedi(){  
    let username = document.getElementById('username').value
    let password = document.getElementById('password').value
    document.getElementById('usr').innerText="Questo è l'username che hai inserito: " + username
    document.getElementById('pass').innerText="Questa è la password che hai inserito: " + password
}

Fatto. Ora abbiamo la nostra funzione accedi() che legge i valori inseriti nei campi di testo (username e password) e li memorizza nelle variabili username e password.


Le variabili JavaScript

Tutti i linguaggi di programmazione vengono aggiornati e così si evolvono. Ciò è ovviamente successo anche al JS. Oggi parleremo delle variabili e delle loro differenze.

Prima di iniziare premettiamo che in informatica esistono due macro-categorie dei linguaggi in base alle variabili. Alcuni linguaggi, come il C o il C++, hanno le variabili di un solo tipo: interi, booleani, stringhe, ecc. Quando dichiari una variabile in C++ devi dirgli che tipo di variabile è: se crei una variabile int, non potrai che dargli valori interi, così se dichiari una variabile booleana, potrai solo assegnargli un valore vero o falso. Nel JavaScript, come in altri linguaggi, le variabili hanno il tipo che vogliamo noi: (possiamo dichiararla con valore 1 e poi assegnargli 'ciao').

Come dicevo all'inizio, il linguaggio JS si è evoluto nel tempo: inizialmente le variabili si dichiaravano con l'attributo var. Successivamente si introdusse l'attributo let per le variabili e il const per le costanti, anche se ovviamente il var è ancora funzionante e presenta alcune differenze rispetto al let, come oggi vedremo.

        function areaTriangolo(){
            let tipo='ret', b=10, h=5
            console.log(a)
            if(tipo=='ret'){
                let a=b*h/2
                console.log(a)
            }
            console.log(a)
        }
                 

                     CONSOLE:
    > Uncaught ReferenceError: a is not defined
    > 7.5
    > Uncaught ReferenceError: a is not defined

Come vediamo, la variabile let è definita solo all'interno dell'if, non esiste prima e dopo. Ora però facciamo la stessa cosa con var.

        function areaTriangolo(){
            var tipo='ret', b=10, h=5
            console.log(a)
            if(tipo=='ret'){
                var a=b*h/2
                console.log(a)
            }
            console.log(a)
        }
                

                    CONSOLE:

     > undefined
     > 7.5
     > 7.5

Incredibilmente, la variabile definita con var è disponibile anche dopo l'if, e soprattutto prima della sua dichiarazione con valore undefined.

                    function addizione(){
                        let a=5, b=3
                        console.log(a)
                        let x=x/2
                        console.log(x)
                        console.log(a)
                    }
                             

                    CONSOLE:

    > 5
    > Uncaught ReferenceError: Cannot access 'x' before initialization

L'avete notato? Una volta che la variabile definita con let ha trovato un errore non ha eseguito i successivi passaggi. Proviamo ora con var:

    function addizione(){
        let a=5, b=3
        console.log(a)
        var x=x/2
        console.log(x)
        console.log(a)
    }
                             

                    CONSOLE:

    > 5
    > NaN
    > 5

Come era prevedibile, con la variabile dichiarata con var, lo script non si è fermato, ma è andato avanti.

In conclusione, vi consiglio di usare le variabili let, in modo da potervi accorgere tempestivamente di un errore nel codice.
